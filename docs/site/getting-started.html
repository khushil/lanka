Getting Started with LANKA

From zero to your first intelligent application in 15 minutes

Prerequisites

Before installing LANKA, ensure you have the following:

Node.js 20+

Required for running the platform

# Check version
node --version

# Install Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

Neo4j 5.0+

Graph database for knowledge storage

# Download and install
wget -O neo4j.deb https://neo4j.com/download-center/
sudo dpkg -i neo4j.deb
sudo systemctl start neo4j

Git 2.34+

Version control system

# Check version
git --version

# Install/update Git
sudo apt-get update
sudo apt-get install git

Installation

Step 1: Clone the Repository

git clone https://github.com/khushil/lanka.git
cd lanka

Step 2: Install Dependencies

npm install

Step 3: Configure Environment

# Copy environment template
cp .env.example .env

# Edit configuration
nano .env

Update the following variables in your `.env` file:

# Neo4j Configuration
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=your-password

# API Configuration  
API_PORT=3000
API_HOST=localhost

# AI/ML Configuration (Optional)
OPENAI_API_KEY=your-api-key
MODEL_PATH=./models

# Security
JWT_SECRET=generate-a-secure-secret
ENCRYPTION_KEY=generate-encryption-key

Step 4: Initialize Database

# Create database schema
npm run db:init

# Seed with sample data (optional)
npm run db:seed

Step 5: Start the Platform

# Development mode with hot reload
npm run dev

# Production mode
npm run build
npm start

Your First LANKA Project

1. Create a New Project

import { Lanka } from '@lanka/core';

// Initialize LANKA client
const lanka = new Lanka({
  apiUrl: 'http://localhost:3000',
  apiKey: 'your-api-key'
});

// Create a new project
const project = await lanka.projects.create({
  name: 'My First Project',
  description: 'E-commerce platform with AI recommendations',
  type: 'web-application'
});

2. Define Requirements

// Add requirements using natural language
const requirements = await lanka.requirements.createBatch([
  {
    title: 'User Authentication',
    description: 'Users should be able to register and login with email and password',
    priority: 'HIGH',
    acceptanceCriteria: [
      'Email validation',
      'Password strength requirements',
      'Remember me functionality'
    ]
  },
  {
    title: 'Product Catalog',
    description: 'Display products with search and filter capabilities',
    priority: 'HIGH',
    acceptanceCriteria: [
      'Product search by name',
      'Filter by category and price',
      'Sort by popularity and price'
    ]
  }
]);

3. Generate Architecture

// AI generates optimal architecture
const architecture = await lanka.architecture.generate({
  projectId: project.id,
  requirements: requirements.map(r => r.id),
  preferences: {
    style: 'microservices',
    database: 'postgresql',
    frontend: 'react',
    deployment: 'kubernetes'
  }
});

console.log(architecture);
// Output: Complete architecture with services, APIs, and data models

4. Generate Implementation

// Generate code for each service
const services = await lanka.development.generateServices(architecture);

for (const service of services) {
  console.log(`Generated ${service.name}:`);
  console.log(`- Controllers: ${service.controllers.length}`);
  console.log(`- Models: ${service.models.length}`);
  console.log(`- Tests: ${service.tests.length}`);
  
  // Export generated code
  await lanka.development.export(service, `./output/${service.name}`);
}

5. Generate Tests

// Automatic test generation
const testSuite = await lanka.testing.generateSuite({
  services: services.map(s => s.id),
  coverage: {
    unit: true,
    integration: true,
    e2e: true
  },
  edgeCases: true
});

// Run tests
const results = await lanka.testing.run(testSuite);
console.log(`Test Results: ${results.passed}/${results.total} passed`);

CLI Usage

LANKA includes a powerful CLI for common operations:

# Initialize a new project
lanka init my-project

# Add requirements from file
lanka requirements add requirements.yaml

# Generate architecture
lanka architecture generate --style microservices

# Generate code
lanka develop --service user-service

# Run tests
lanka test --coverage

# Deploy to cloud
lanka deploy --env production

# View project status
lanka status

Docker Setup (Alternative)

# docker-compose.yml
version: '3.8'

services:
  lanka:
    image: lanka/platform:latest
    ports:
      - "3000:3000"
    environment:
      - NEO4J_URI=bolt://neo4j:7687
      - NEO4J_USER=neo4j
      - NEO4J_PASSWORD=lanka123
    depends_on:
      - neo4j
  
  neo4j:
    image: neo4j:5.0
    ports:
      - "7474:7474"
      - "7687:7687"
    environment:
      - NEO4J_AUTH=neo4j/lanka123
    volumes:
      - neo4j_data:/data

volumes:
  neo4j_data:
# Start with Docker Compose
docker-compose up -d

# View logs
docker-compose logs -f

# Stop services
docker-compose down

Troubleshooting

Neo4j connection failed

Check that Neo4j is running:

sudo systemctl status neo4j

Verify connection details in `.env` file match your Neo4j configuration.

Port 3000 already in use

Change the port in `.env`:

API_PORT=3001

Or kill the process using port 3000:

lsof -i :3000
kill -9 [PID]
npm install fails

Clear npm cache and retry:

npm cache clean --force
rm -rf node_modules package-lock.json
npm install
